# pi-web 0.2.0

0.2.0 is about making pi-web feel more like a persistent, mobile-first working environment while keeping the README promise intact: minimal, local/Tailscale-friendly, and close to the spirit of the core pi agent.

## Highlights

### Session workspace improvements

- Added session marker buckets
- Refined session marker interactions
- Simplified session markers and UI state syncing
- Added color-aware session tab and drawer filtering
- Improved the session drawer pinning workflow
- Added a current-session bucket menu
- Added session deletion
- Removed a redundant session drawer filter

### Continuity and navigation

- Persisted composer state across refreshes
- Improved reload robustness during startup
- Auto-reload open tabs when service worker updates
- Improved session tab switch rendering performance
- Added scoped keyboard shortcuts
- Added `/clear-tab`
- Fixed jump-to-latest button behavior

### Message rendering

- Added folder creation and thinking message support
- Collapsed thinking cards in compact mode
- Improved thinking and stop-reason test coverage

### Reliability, tests, and releases

- Added a PR workflow that runs tests on pull requests
- Fixed Playwright report artifact upload
- Updated GitHub Actions versions and removed warning suppression
- Stabilized the release workflow across macOS/headless browser constraints
- Published release packages without requiring browser installation

## Intent

The main product shift is subtle: pi-web is still intentionally small, but it now handles longer-lived work better. Sessions are easier to organize, refreshes are less disruptive, mobile navigation is more capable, and release/test automation is more dependable.
